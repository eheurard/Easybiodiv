Rendu des marqueurs pie chart sur la carte

La dette écologique (Lbiodiv) est affichée comme des points pie chart sur la carte. Quelle approche de rendu ?

A

SVG custom markers (MapLibre Marker API)

Chaque point = un élément SVG injecté comme marqueur MapLibre. Le camembert est dessiné avec des <path> SVG, la taille scale avec le Lbiodiv. Tooltips HTML natifs au survol.

Avantages

  • Vanilla JS pur — cohérent avec le projet
  • Facile à styliser et maintenir
  • Tooltips DOM natifs

Inconvénients

  • Peut ralentir avec >500 points

Recommandé — nombre de points < 100 sur ce dashboard.

B

Canvas → PNG → Symbol layer MapLibre

Chaque pie chart est pré-rendu sur un <canvas> offscreen, converti en dataURL PNG, puis injecté comme icône dans un layer MapLibre symbol.

Avantages

  • Très performant (WebGL rendering)
  • Supporte des milliers de points

Inconvénients

  • Plus complexe à implémenter
  • Tooltips moins naturels
  • Surqualifié pour ce cas d'usage
C

SVG overlay D3-style (couche au-dessus de MapLibre)

Un <svg> est superposé à la carte. Les positions sont converties en pixels via la projection MapLibre. D3 dessine les pie charts sur ce calque.

Avantages

  • Très flexible visuellement
  • Pas de dépendance D3 — géométrie SVG manuelle possible

Inconvénients

  • Sync coordonnées complexe au zoom/pan
  • Plus de code, plus fragile
