Comment charger les données de comparaison ?

Les 5 métriques proviennent de fonctions différentes dans views.py. Comment les exposer au client ?

A

Endpoint dédié /api/company/<pk>/compare/

Une nouvelle fonction _get_comparison_data(company) agrège les 5 métriques en un seul appel JSON. Le JS fait 1 requête par entreprise sélectionnée.

Pour

  • 1 seul appel réseau par entreprise
  • Logique centralisée côté serveur
  • Facile à étendre

Contre

  • Nouvelle route à ajouter

⭐ Recommandé

B

Réutiliser les endpoints existants

Pour chaque entreprise, le JS appelle en parallèle /mesure-empreinte/, /dette-ecologique/ et /company/, puis en extrait les métriques utiles.

Pour

  • Aucun nouvel endpoint
  • Réutilise le code existant

Contre

  • 3+ appels réseau par entreprise
  • Charge des données inutiles
  • JS plus complexe
C

Tout server-side (rechargement de page)

Un formulaire GET avec les 2 entreprises et les métriques choisies. Chaque changement recharge la page, Django calcule tout et injecte les données dans le template.

Pour

  • Très simple, pas de JS complexe

Contre

  • Rechargement à chaque sélection
  • UX moins fluide
  • Ne suit pas le pattern existant
