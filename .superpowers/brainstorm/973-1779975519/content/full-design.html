Notes d'implémentation
1
Combobox entreprise (header) — Input texte avec filtre JS sur la liste des entreprises. Au choix, un fetch('/dashboard/api/company/<id>/') recharge les données sans reload. La barre de recherche actuelle est retirée du header sur cette page.
2
KPI cards — Données servies par l'endpoint JSON : asset_count, country_count, commodity_count, region_count. Mise à jour JS du DOM à chaque changement d'entreprise.
3
Carte MapLibre GL JS — Chargée via CDN. Tuiles depuis OpenFreeMap (gratuit, pas de clé API). Markers placés aux coordonnées lat/lng des actifs. Popup au clic : nom de l'actif, pays, commodité. Markers mis à jour via map.getSource().setData() sans recréer la carte.
4
Exposition par pays — Liste scrollable des pays avec leur nombre d'actifs et les commodités associées (tags). Données issues de l'endpoint JSON : countries: [{name, flag, asset_count, commodities: [{name, count}]}].
5
Endpoint Django — Nouvelle vue FBV company_data(request, pk) dans dashboard/views.py, route api/company/<int:pk>/. Requête via Ownership → Asset → agrégation par pays/commodité. Retourne JSON, décorée @require_GET.
